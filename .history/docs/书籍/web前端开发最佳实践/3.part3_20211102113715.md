# 6. 高性能维护的 CSS

```js
1. 如何高效地组织CSS代码
通用类(common.js)、业务类、default.css(重置)
2. 使用CSS Reset : 统一浏览器显示效果
  * 配置所有的东西
3. 给css样式定义排序 块级-属性-状态
  1. 按类型分组进行排序---影响布局的排在前面，不影响的排在后边
  2. 按字母排序
  3. 按定义长度排序---使用较少
  4. 合理使用CSS的权重:提高代码的重用性
     1. css样式中尽量不要使用ID选择器
     2. 减少子选择器的层级
     3. 使用组合的CSS类选择器---多组合少继承
  5. CSS代码的兼容性：如何兼容IE浏览器
     1. hack
     2. 分离样式兼容代码
  6. em px 还是%
     字体相对单位：em rem ex ch
     视图相对单位：vw vh vmin vmax
     1. 尽量设置相对尺寸em %
     2. 只有在可预知元素尺寸的情况下才使用绝对尺寸---固定的宽高
     3. 使用em设置字体大小
```

# 7. 高性能的 CSS

```js
1. 使用高效的CSS选择器
   1. 避免使用通配符
   2. 避免使用标签选择器及单个属性选择器作为关键选择器
   3. 不要在ID选择器前使用标签名
   4. 尽量不要在选择符中定义过多的层级，最好不要超过三层
2. CSS相关的图片处理
   1. 不要设置图片的尺寸
   2. 使用CSS Sprite技术
3. 减少CSS的代码量
   1. 定义简洁的CSS规则
   2. 合并CSS规则
   3. 定义简洁的属性值
   4. 合并相同的定义
   5. 删除无效的定义
4. 其他CSS高性能实践
   1. 避免使用"@import"
   2. 避免使用IE浏览器独有的样式：图片滤镜和CSS表达式
```
