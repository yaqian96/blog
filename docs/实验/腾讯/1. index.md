# 谈谈你对 dns-prefetch 的理解
```js
使用方式
<link rel='dns-prefetch' href='mp.weixin.qq.com'>
默认加载条件
所有的a标签的href都会自动启动DNS Prefetching（a标签href带的域名不需要在head里面加上link手动设置）
a标签默认启动在HTTPS不起作用，这就需要使用meta的http-equiv属性来强制启动；
<meta http-equiv='x-dns-prefetch-control' content='on'>

1. DNS Prefetching提前进行域名解析，后续加载省去相应的解析时间。a标签的href是可以在chrom/firefox/高版本的ie，但是在HTTPS下面不起作用，需要使用meta的http-equiv属性来强制启动；
2. 这是对DNS的提前解析，并不是css/js之类的文件缓存
3. 如果直接做了JS的重定向，或者在服务端做了重定向，没有在link里面手动设置，是不起作用的。
4. 这个对于什么样的网站更有作用呢，类似淘宝这种引用了大量其他域名的资源，如果你的网页，所有的资源都在一个域名下，那么基本基于没有什么作用，因为DNS Chrom再访问你的网站就帮你缓存了
```
# 内部属性 [[Class]] 是什么？
```js
1. 对象的[[class]]属性
多数情况下，对象的内部[[class]]属性和创建该对象的内建原生构造函数相对应
2. 基本类型的[[class]]属性
虽然Null()和Undefined()这样的原生构造函数并不存在，但是内部[[class]]属性仍然是“Null”和“Undefined”。
```
# 域名收敛和域名发散
```js
1. 域名收敛
   解决问题：将手机端的数据进行
2. 域名发散 DS
   1. 设置方法
    使用CNAME进行设置，Canonical Name。设置alias
   2. DS与CDN的区别
   DS: 为了突破浏览器对连接数的限制，而将资源放到不同的domains 上的
   CDN: 就是将你的资源放到 全国 或者说 全世界各个地方的服务器上的.(当然，他本身也是利用DS的技术)
   3. SPDY
   在不增加域名的情况下，解除最大连接数的限制
```
