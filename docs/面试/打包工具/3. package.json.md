```js
// package.json
{
  "name": "mobile-webview", //包名
  "version": "1.1.2",   //版本号，【主版本 . 次要版本 . 补丁版本】
  "description":"项目的描述" //项目的描述
  "style": "normalize.css", //提供样式所在的文件位置
  "private": true,   //防止应用程序/软件包被意外发布到 npm 上
  "fastgeek": "vue",
  "main": "normalize.css", //主文件的入口
  "homepage": "http://111" //包的官网
  "keywords":""   //此属性包含与软件包功能相关的关键字数组。
  "repository": {
    "type": "git",
    "url": "git://",
  }  //包代码仓库的信息，包括type和URL，type可以是git或svn，URL则是包的Repo地址。
  "scripts": {
    "serve:dev": "vue-cli-service serve --mode development --fix",
  }, //可以定义一组可以运行的 node 脚本。
  "dependencies": {
    "@antv/f2": "^3.7.7",
  },
  "devDependencies": {
    "@babel/plugin-proposal-optional-chaining": "^7.14.2",
    "@vue/cli-plugin-babel": "~4.3.0"
  },
  "engines": {
    "node": ">= 6.0.0",
    "npm": ">= 3.0.0",
    "yarn": "^0.13.0"
  } //设置此软件包/应用程序要运行的 Node.js 或其他命令的版本。
  "browserslist": [
    "> 1%",
    "last 2 versions",
    "not ie <= 8"
  ] //用于告知要支持哪些浏览器（及其版本）。 Babel、Autoprefixer 和其他工具会用到它，以将所需的 polyfill 和 fallback 添加到目标浏览器。
  "husky": {
    "hooks": {
      "pre-commit": "lint-staged",
      "commit-msg": "validate-commit-msg"
    }
  },
  "lint-staged": {
    "*.{js,jsx,vue}": [
      "vue-cli-service lint",
      "git add"
    ]
  }, //husky 是一个 Git Hook 工具。主要实现提交前 eslint 校验和 commit 信息的规范校验，也可以避免多人合作时代码格式化不统一造成的冲突；
  "author": "huanglu",
  "license": "MIT" //许可证
  //contributors - 包的其他贡献者。
}

```

# version 版本

```js
1. 使用~表示版本范围
~2.3.4 //2.3.4 <= version < 2.4.0
~2.3   //2.3.0 <= version < 2.4.0
~2.3   //2.0.0 <= version < 3.0.0
2. 使用^表示版本范围
^1.3.4 //1.3.4 <= version < 2.0.0
^0.2.3 //0.2.3 <= version < 0.3.0
^0.0.3 //0.0.3 <= version < 0.0.4
3. 语义版本号 X.Y.Z
  1. 如果只是修复bug，需要更新Z位。
  2. 如果是新增了功能，但是向下兼容，需要更新Y位。
  3. 如果有大变动，向下不兼容，需要更新X位
```
